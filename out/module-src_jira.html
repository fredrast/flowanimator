(inner) recursiveFetchFromJira(ul, id, token, parameters, startAt, fieldName, values)

Recursive function making multiple requests to the Jira REST API to fetch a complete list of items in situations where each request only retrieves a subset of all items.

Parameters:
Name Type Description
ul

Url to the Jira REST API resource (including possible cors proxy url)
id

User id for logging to the Jira server, given by user
token

Password or API token for logging to the Jira server, given by user
parameters

Possible parameters to be included in the HTTP request
startAt

The index of the first item to be retrieved
fieldName

The name of the field in the response JSON that contains the list of the items that we want to retrieve. This can be different for different API resources, e.g. when retrieving boards, the field is called "values" but when retrieving issues, the field is called "issues".
values

Holds the list of items retrieved through the calls to the API. This list is passed as a parameter and added to in each subsequent API call.
